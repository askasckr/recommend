# recommend
A simple portfolio recommendation tool ``` RecsApplication ```.

The following libraries are used:
```
h2 embedded database
project lombok
guava libs (RateLimiter)
apache commons
hibernate
springboot 2.x
Mockito
SpringBootTest
```
### Assumptions:
-----------------

A new ```LOOKUP.INVESTMENT_CATEGORY``` or ```LOOKUP.INVESTMENT_RISK```  record can be added to the database independent of the records ```RECOMMEND.PREDEFINED_PORTFOLIO```  table, but a validation takes place during the updates or save into ```RECOMMEND.PREDEFINED_PORTFOLIO```  table to make sure the percents sums up to 100.

**Note:** All the requests expect Client-Id in header to distinguish the clients. More than two concurrent client requests result in HTTP 429 Too Many Requests response status code.

The below **seed data** is populated at app startup (look at ``` SeedDataService ```):

**DB Schema** looks like below:
```
    create schema lookup;
    create schema recommend;
    
    drop table lookup.investment_category if exists
    drop table lookup.investment_risk if exists  
    drop table recommend.predefined_portfolio if exists

    create table lookup.investment_category (
       id integer generated by default as identity,
        created timestamp not null,
        created_by varchar(50) not null,
        modified timestamp,
        modified_by varchar(50),
        display_order integer not null,
        info varchar(255),
        name varchar(20),
        primary key (id)
    )

    create table lookup.investment_risk (
       id integer generated by default as identity,
        created timestamp not null,
        created_by varchar(50) not null,
        modified timestamp,
        modified_by varchar(50),
        info varchar(255),
        level integer not null check (level>=1 AND level<=10),
        primary key (id)
    )
    
    alter table lookup.investment_category 
       add constraint UK_poaco92efxypo9j7seumfob4u unique (display_order)
       
    alter table lookup.investment_category 
       add constraint UK_kx8o5f7kcvlp1qh3dguygpwpe unique (name)

    alter table lookup.investment_risk 
       add constraint UK_k3qc7almcqun62ff0aocqupko unique (level)

    create table recommend.predefined_portfolio (
       id integer generated by default as identity,
        created timestamp not null,
        created_by varchar(50) not null,
        modified timestamp,
        modified_by varchar(50),
        percent float not null check (percent>=0 AND percent<=100),
        investment_category_id integer not null,
        investment_risk_id integer not null,
        primary key (id)
    )

    alter table recommend.predefined_portfolio 
       add constraint UK53w1nebdt21xsnn7yug7xm3ww unique (investment_category_id, investment_risk_id)

    alter table recommend.predefined_portfolio 
       add constraint FKj7ch35liqjv44013dwdwhv7q8 
       foreign key (investment_category_id) 
       references lookup.investment_category

    alter table recommend.predefined_portfolio 
       add constraint FK3yfajg46jw4i5fylmng3norix 
       foreign key (investment_risk_id) 
       references lookup.investment_risk

```
